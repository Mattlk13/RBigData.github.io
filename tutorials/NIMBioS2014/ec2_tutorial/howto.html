How to Launch Our Public Amazon Machine Image (AMI) as an Elastic Compute Cloud (EC2) Instance

  1. From the Amazon Web Services homepage , select "AWS Management Console" from the "My Account/Console" drop down menu.

  2. Enter your AWS account credentials and sign in.

  3. Select "EC2: Virtual Servers in the Cloud" from the selection list.

  4. Change the current region to US East (N. Virginia) from the right side of the top navigation menu (if it is not already selected). Our provided Amazon Machine Images are located in the US East datacenter.

  5. Select "AMIs" from the left-hand selection list, under the "Images" section.

  6. From the "Filter" Bar, under the drop down menu labeled "Owned by me", select "Public images".

  7. Enter the AMI ID which we have provided to you (ami-6727350e) into the search area. Select the AMI and then click "Launch".

  8. Select the type of EC2 instance to run. Here we choose "General purpose".

  9. Now select the specific instance type. Here we choose "m3.2xlarge", an 8 vCPU instance. Afterwards, select "Review and Launch".

  10. Select "Edit security groups" in the lower right hand corner.

  11. Click "Add Rule".

  12. Select "HTTP" from the "Type" drop down menu.

  13. Click "Review and Launch".

  14. Now click "Launch".

  15. For users who only wish to use Rstudio, you will not need to worry about setting up an RSA key pair during this step.
    Select "Proceed without a key pair", click the checkbox, and then click "Launch Instances".

  16. Check the launch log of your instances if you wish. When you are finished, click "View Instances".

  17. Select the running instance which we have created. In the bottom portion of the screen, additional details are displayed.
    Copy the "Public DNS" field. We will need this in the next step to access the Rstudio web interface.

  18. Paste the "Public DNS" from the previous step into the web browser URL area.
    If your instance is currently running, then you should be greeted with the Rstudio login page.
    Enter "rstudio" for both the username and password to login as the "rstudio" user.

  19. Congratulations, you have now successfully set up Rstudio on your own server in the cloud.
